# QA（高频问答）

- **Q：[[IDaaS/CIAM/index|CIAM]] 的计费模式是怎样的？**  
  A：阿里云 CIAM 采用月活用户数（MAU，Monthly Active User）计费模式，仅对实际活跃使用的用户收费，避免为陈旧或非活跃账号支付冗余成本。该模式兼顾公平性与灵活性，适用于用户活跃度波动较大的业务场景。详情请参见 [[产品对外文档/定价说明|定价说明]]。

- **Q：CIAM 是否支持已有身份系统的平滑迁移？**  
  A：支持。CIAM 提供完整的**统一身份整合能力**，包括账号去重（疑似重复账号识别与人工确认合并）、僵尸账号清理、多源身份数据同步（支持懒加载与增量同步），可协助企业逐步整合历史身份孤岛，实现“老系统不停、新体系渐进”的平滑转型。

- **Q：如何提升用户注册/登录体验，降低流失率？**  
  A：CIAM 提供多项体验优化能力：  
  - **分步注册**：按业务流程分阶段收集信息（如登录时仅验身份、下单时填地址、付款时补实名），避免首屏表单过长导致约20%用户放弃；  
  - **多因子认证与社交登录**：支持微信、支付宝等主流社交账号一键登录，同时提供MFA、行为风控驱动的动态认证策略；  
  - **高可用架构**：应对零售、旅游等行业高峰流量，保障关键时段服务不降级；  
  - **自助服务门户**：覆盖密码找回、凭证更换、日志查看、账号删除等全生命周期操作，大幅降低人工客服压力。

- **Q：CIAM 如何满足《个人信息保护法》《数据安全法》等合规要求？**  
  A：CIAM 内置**同意收集与条款管理**能力，支持条款版本统一发布、用户签署记录全链路留存、同意状态跨平台同步，彻底解决“条款孤岛”问题；同时所有数据处理均遵循最小必要原则，结合阿里云整体安全合规基线（等保三级、ISO 27001、SOC2 等），确保客户面向终端用户的身份服务合法合规。相关能力详见 [[产品对外文档/合规指南|合规指南]]。

- **Q：CIAM 是否提供开箱即用的登录页和用户门户？**  
  A：是。CIAM 支持两种集成方式：  
  - **低代码接入**：直接使用阿里云提供的标准化通用登录页与用户自助服务门户，开箱即用，快速上线；  
  - **深度定制**：通过开放 API/SDK 与企业自有前端无缝集成，灵活适配品牌 UI 与业务逻辑。两种方式均默认启用全部安全策略（如防暴力破解、IP限频、异常登录拦截等）。

- **Q：当用户在异地、陌生设备频繁登录时，系统如何响应？**  
  A：CIAM 风控引擎基于用户历史行为建模（常用IP、设备指纹、访问时段、地理位置等），实现千人千面的风险识别。检测到异常登录请求时，自动触发分级响应策略，例如：  
  - 中风险：弹出短信/邮箱二次验证；  
  - 高风险：强制跳转至MFA认证页并通知用户；  
  - 极高风险：临时锁定账号并推送告警至管理员控制台。  
  所有策略均可在控制台可视化配置，并支持回调通知供业务侧联动处置。

- **Q：CIAM 是否支持私有化部署？**  
  A：阿里云 IDaaS CIAM 当前为**公有云托管服务**，不提供私有化部署版本。其核心优势（如自动热更新漏洞补丁、实时同步最新安全策略、弹性扩缩容能力）均依托阿里云公共云基础设施实现。如需混合云或专属云方案，建议联系阿里云售前架构师评估 [[产品对外文档/混合云集成方案|混合云集成方案]]。
