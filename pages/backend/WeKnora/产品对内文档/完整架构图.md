# 完整架构图

- **Web 前端**（含 Web UI）：通过 HTTP/HTTPS 接入 API 网关
- **IM 客户端**：支持飞书、企业微信、Slack、Telegram、钉钉、微信、Mattermost 等，通过 Webhook 或 SDK 接入 IM 接入网关；部分通道（如钉钉/飞书/企微）也支持通过 API 网关 统一代理
- **桌面客户端**（Desktop App）：基于 Wails 构建，通过 IPC 与 API 网关 通信
- **CLI 工具**：提供诊断（`doctor/cmd`）、分块调试（`chunk/cmd`）、链接管理（`link/cmd`）、配置管理（`config/`）等运维能力
- **MCP 客户端**：通过 MCP 适配器 接入系统，实现标准化模型控制协议交互

## 核心服务层

- **认证中心**（`auth/cmd`）：统一处理 JWT/OAuth2/SSO 认证与权限校验，为所有入口提供鉴权支撑  
- **会话管理**（`session/cmd`）：维护用户级长连接状态、上下文生命周期，并向 Agent 编排引擎 注入实时会话上下文  
- **知识库服务**（`kb/cmd`）：管理知识库元数据（存储于 MySQL/PostgreSQL）、访问控制、版本快照及同步策略  
- **文档解析服务**（`doc/cmd + docreader/`）：驱动全链路文档处理流水线，包括格式识别、内容提取与结构化预处理  
- **Agent 编排引擎**（`agent/cmd`）：核心调度中枢，协调记忆管理（`agent/memory`）、工具调用（`agent/tools/`）、审批工作流（`agent/approval`）与 Token 级编排（`agent/token`）  
- **混合检索服务**（`search/cmd + retriever/`）：统一调度多模态检索能力，融合向量、图、关系型与全文检索结果  
- **聊天管道引擎**（`chat_pipeline/service`）：构建上下文（`context/cmd`）、管理会话状态机（`models/chat/`）、输出流式响应（SSE）  
- **技能调度器**（`skills/ + agent/skills`）：动态加载并执行原子化技能（如代码解释、表格分析、公式推导），与 LLM 服务集群 和 Embedding 服务 协同  

## 数据处理与 AI 能力层

- **文档处理流水线**
  - **多模态解析器**（`docreader/parser/`）：原生支持 PDF/Word/Excel/PPT/Markdown/HTML/Notion/飞书/语雀等格式，集成 OCR 增强与公式识别能力  
  - **智能分块器**（`docreader/splitter/`）：支持语义分块、代码分块、表格保留、跨页逻辑合并等高级策略  
  - **ASR/VLM 模型**（`models/asr/ + vlm/`）：集成 Whisper、腾讯云 ASR 及图文理解多模态模型，支撑音视频与图像内容理解  

- **嵌入与重排序能力**
  - **Embedding 服务**：支持 BGE、M3E、腾讯混元向量等主流模型，由 `models/embedding/` 统一管理  
  - **Rerank 服务**：集成 BGE-Reranker、CoRRanker 等模型，对初检结果进行精细化相关性重打分  

- **混合检索引擎**
  - **向量检索**：对接 Qdrant、Milvus、Weaviate、Tencent VectorDB 等向量数据库  
  - **图检索**：基于 Neo4j 构建结构化知识图谱，支持实体关系推理与路径查询  
  - **关系型索引**：利用 PostgreSQL/SQLite 存储结构化元数据与业务关联信息  
  - **全文检索**：依托 Elasticsearch 或 Doris 实现关键词高精度匹配与聚合分析  
  - **本地向量库**：轻量场景下支持 SQLite 内嵌向量索引（兼容 Tencent VectorDB 本地模式）  

- **大模型服务编排**
  - **LLM 服务集群**（`models/llm/`）：统一调度 OpenAI、Ollama、通义千问、腾讯混元、DeepSeek、Claude 等多后端模型，支持流式响应与函数调用（Function Calling）  
  - **多模型协同机制**：按任务类型自动路由至最适模型（如代码→CodeLlama，数学→DeepSeek-Math，中文→Qwen），并支持模型间接力生成  

## 数据源与基础设施层

- **多源数据连接器**
  - 飞书知识库连接器（`connector/notion/` → 实际含飞书适配）
  - 语雀连接器（`connector/yuque/`）
  - Web 抓取器（`web_fetch/`）与网络搜索增强（`web_search/`）
  - 数据库直连（`datasource/connector/`）：支持 MySQL、PostgreSQL、SQL Server、Oracle 等 JDBC 数据源
  - API 数据源：通过通用 REST/GraphQL 适配器接入第三方系统  

- **持久化存储体系**
  - **原始文档存储**：MinIO/S3/OSS/本地文件系统（FS），承载原始二进制文件  
  - **结构化知识图谱**：Neo4j 图数据库（`repository/memory/neo4j`），用于实体关系建模与推理  
  - **向量数据库集群**：Milvus/Qdrant/Weaviate/Tencent VectorDB（`retriever/milvus/...`）  
  - **关系型数据库**：PostgreSQL（主业务库）、SQLite（边缘/单机场景）  
  - **知识库元数据**：MySQL/PostgreSQL（`kb/cmd` 元数据管理）  

- **可观测性基础设施**
  - **统一日志中心**：基于 Langfuse + 自研 Tracing 的全链路日志采集与可视化（`logger/`）  
  - **链路追踪**：Langfuse 集成，覆盖从 IM 接入网关、API 网关、Agent 引擎到 LLM 调用的完整 Span  
  - **指标监控**：Prometheus + Grafana，采集服务延迟、Token 吞吐、检索召回率、模型负载等核心 SLO 指标  
  - **事件总线**（`event/`）：基于 Kafka/RabbitMQ 实现异步解耦，支撑文档变更通知、审批事件广播、缓存刷新等场景  

## 运维与扩展能力

- **CLI 工具链**：提供 `doctor/cmd`（健康诊断）、`chunk/cmd`（分块策略验证）、`link/cmd`（外部数据源绑定）、`config/`（运行时配置热更新）等生产级运维能力  
- **部署支持**
  - Helm Chart（`helm/`）：面向 Kubernetes 生产环境的一键部署方案  
  - Docker Compose（`docker/`）：适用于测试与中小规模私有化部署  
  - 桌面版打包（`cmd/desktop/`）：支持 Windows/macOS/Linux 桌面客户端离线安装与自动更新  
- **配置与服务治理**
  - **配置中心**：Consul/Etcd，管理模型参数、分块策略、检索权重等动态配置  
  - **服务注册发现**：Nacos/Consul，支撑微服务自动扩缩容与故障转移  
  - **告警中心**：Alertmanager，对接 Prometheus 指标与自定义业务事件（如知识库同步失败、模型超时）  

## 关键跨层协作机制

- **事件驱动上下文注入**：IM 接入网关通过事件总线将用户行为、消息上下文实时推送至会话管理模块，再注入 Agent 编排引擎，保障对话连贯性  
- **动态技能加载**：技能调度器按需从插件目录或远程仓库加载技能包（含描述、Schema、执行逻辑），无需重启服务  
- **混合检索结果融合**：向量/图/关系/全文四路检索结果经 Rerank 重排序后，由融合策略（RRF、Weighted Sum、LLM-based Fusion）生成最终 Top-K 结果集  
- **生成式响应流式交付**：聊天管道引擎将 LLM 流式输出、工具执行中间结果、知识引用片段统一组装为 SSE 响应，经 API 网关或 IM 接入网关实时推送至前端  
- **全链路可观测性贯通**：Langfuse Tracing ID 贯穿用户请求 → 网关 → 认证 → 检索 → Agent → LLM → 响应全过程，支持根因定位与性能归因分析
