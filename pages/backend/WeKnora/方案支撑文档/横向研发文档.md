# 横向研发文档

*   接入指引  
    WeKnora 提供统一 CLI（`wk`）与 RESTful API 两种接入方式，推荐企业级集成优先使用 API 方式，CLI 更适用于本地开发调试与自动化脚本。所有对外能力均通过 `cmd/server/` 启动的 HTTP 服务暴露，核心路由由 `internal/router/` 统一注册，支持 OpenAPI 3.0 规范，文档详见 API 文档 和 [[docs/wiki/API参考|API 参考]]。身份认证采用 OAuth2 + JWT 模式，对接逻辑位于 `cli/cmd/auth/` 与 `internal/handler/auth/`，支持企业自有 Identity Provider（IdP）通过 OIDC 协议集成。

*   产品对接方案细节  
    - **多模型服务编排**：通过 `internal/models/provider/` 抽象统一模型调用层，支持 LLM/Embedding/Rerank/VLM/ASR 五类模型动态注册与路由；各模型实现按厂商/协议隔离（如 `internal/models/embedding/openai/`, `internal/models/rerank/cohere/`），可通过配置文件 `config/model.yaml` 或环境变量热切换。  
    - **混合检索引擎**：`internal/application/repository/retriever/` 下提供 10+ 种向量/图/关系型检索后端适配器（Milvus、Qdrant、Neo4j、Doris、Tencent VectorDB、Weaviate 等），支持运行时声明式组合（如“向量初筛 + Neo4j 图关系精排 + Doris 属性过滤”），检索策略由 `internal/application/service/retriever/` 统一编排。  
    - **IM 集成**：`internal/im/` 目录实现飞书、企微、Slack、Telegram、钉钉、Mattermost、微信等主流 IM 的双向消息收发、事件订阅与卡片渲染，各 connector 独立封装（如 `internal/im/feishu/`, `internal/im/wecom/`），支持通过 `config/im.yaml` 启用/禁用及配置 Webhook 地址与密钥。  
    - **MCP（Model Context Protocol）标准化工具调用**：`internal/mcp/` 与 `cli/internal/mcp/` 实现 MCP v0.1 兼容的工具发现、参数校验与执行代理，前端可通过 `frontend/src/api/skill/` 调用已注册技能，后端技能插件需遵循 `mcp-server` 接口规范，示例见 技能示例集。  
    - **文档解析与分块**：`docreader/` 与 `internal/infrastructure/docparser/` 支持 PDF/DOCX/PPTX/Markdown/HTML 等 15+ 格式解析，`internal/infrastructure/chunker/` 提供语义分块（LLM-based）、规则分块（Markdown heading）、表格结构保留等多种策略，可按知识库粒度配置。

*   产品对接范围  
    | 类别 | 覆盖能力 | 对接路径/模块 | 备注 |
    |--------|------------|----------------|------|
    | **知识接入** | 多源文档上传、解析、分块、元数据提取 | `cli/cmd/doc/`, `internal/datasource/`, `docreader/`, `internal/infrastructure/chunker/` | 支持 S3/OSS/本地文件/IM 消息附件/Notion/语雀/飞书多维表格等 |
    | **检索增强** | 向量检索、图谱关系检索、关键词检索、混合排序 | `internal/application/repository/retriever/`, `internal/application/service/retriever/` | 可组合多种检索器，支持自定义 rerank 插件 |
    | **智能体编排** | Agent 生命周期管理、记忆存储、工具调用、审批流 | `internal/agent/`, `internal/agent/approval/`, `internal/agent/tools/`, `internal/agent/memory/` | 内置审批、知识库问答、文档摘要、代码解释等基础技能 |
    | **模型服务** | LLM/Embedding/Rerank/VLM/ASR 统一调度与降级 | `internal/models/provider/`, `internal/models/llm/`, `internal/models/embedding/` 等子目录 | 支持多模型并行请求、fallback、负载均衡、Token 限流 |
    | **IM 集成** | 消息收发、事件监听、交互卡片、会话上下文同步 | `internal/im/`, `internal/handler/session/`, `cli/cmd/chat/` | 各 IM connector 独立部署，支持灰度发布与故障隔离 |
    | **安全与治理** | OAuth2 认证、RBAC 权限控制、审计日志、敏感词过滤 | `cli/cmd/auth/`, `internal/middleware/`, `internal/handler/auth/`, `internal/event/` | 权限模型基于组织（Tenant）、知识库（KB）、会话（Session）三级粒度 |
