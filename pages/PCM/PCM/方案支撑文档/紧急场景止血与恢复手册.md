# 紧急场景止血与恢复手册

在各类异常和灾难场景下，PCM 提供了相应的容错逻辑与应急恢复手段，以保障业务连续性：

| 场景 | SDK 行为 | 业务影响 |
| --- | --- | --- |
| 新部署时 PCM Core 还未 ready | 将入参作为返回 | 无影响（Core 未禁用老 AK） |
| 运行时 PCM Core 挂了 | 返回上次获取的老凭证（未在窗口期末尾） | 无影响 |
| 产品独立升级，PCM 未 ready | 将入参作为返回 | 无影响 |
| PCM 和应用都挂了需重拉（SDK 缓存未丢失） | 返回上次获取的老凭证 | 无影响 |
| PCM 和应用都挂了需重拉（SDK 缓存丢失） | **需先恢复 PCM 或使用老凭证应急脚本** | **业务中断** |

## 组件级降级与止血机制

### PCM SDK / CLI 容错降级
当 PCM 初始化服务异常或报错时，SDK 具备自动容错降级能力以保障应用可用：
- 若本地存在缓存，将返回最近一次从服务端获取的凭证。
- 若无缓存，将入参（原始底表 AK）作为凭证返回，确保应用能够继续运行。

### PCM Core 降级保护
当 PCM Core 发生宕机等故障时，系统会触发降级保护机制：
- 末期过期老凭证行为暂停。
- SDK 将返回上次获得的老凭证（只要未在窗口期末尾），应用依然可以使用该凭证进行访问，从而为修复 Core 争取止血时间。
