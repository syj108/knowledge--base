# 典型问题排查解决方案

● **问题现象**：用户或内部服务在调用云产品API时请求被拒绝，返回401、403或500等错误状态码。错误信息通常提示AccessKey无效或被禁用（如 `InvalidAccessKeyId`、`Unauthorized`、`The Access Key is disabled`）。
● **适用范围**：PCM（权限控制管理）组件。影响所有依赖AK/SK进行鉴权的云产品网关，包括但不限于OSS、SLS、ASAPI、KMS等。

## 二、排查信息收集
● **必须收集的信息**：报错的AccessKeyId、请求发生时间、调用的云产品名称（OSS/SLS/ASAPI等）、RequestID/TraceID、客户端IP。
● **检查终态的方法**：登录对应网关的日志服务器（如OSS Tengine节点、SLS fcgi_agent节点、ASAPI ApiServer节点）查看本地日志文件，或通过SLS统一日志查询平台检索对应Logstore的访问日志。
● **排查问题步骤及关键日志特征映射**：

| 网关类型 | 日志路径/位置 | 关键特征字段及报错信息 | AK提取字段 | 对应场景 |
| :--- | :--- | :--- | :--- | :--- |
| **OSS** | `/apsara/module_logs/oss_tengine/access_log.*` | `error_code="InvalidAccessKeyId"`<br>`status="403"` | `access_id` | OSS访问被PCM拦截 |
| **SLS (Inner)** | `/apsara/fcgi_agent/ols_operation_2.LOG` | `Status="401"` | `AccessKeyId` | SLS内部访问被PCM拦截 |
| **SLS (Pub)** | `/apsara/sls/fcgi_agent/ols_operation.LOG` | `Status="401"`<br>`ErrorCode="Unauthorized"`<br>`ErrorMsg` 包含 "AccessKeyId is disabled" | `AccessKeyId` | SLS公网/外部访问被PCM拦截 |
| **ASAPI** | `/apsara/cloud/data/asapi/ApiServer#/api-server/logs/asapi-logger/audit.log` | `errorCode="asapi.server.request.parameter.accesskeyid.error"`<br>`errorMessage` 包含 "The Access Key is disabled." | `accessKeyId` | ASAPI网关访问被PCM拦截 |

## 三、解决步骤
### 场景一：确认AK被PCM禁用并恢复
- **适用条件**：通过上述日志排查，确认报错特征符合PCM拦截特征（如日志中明确提示 "The Access Key is disabled" 或 "InvalidAccessKeyId" 且排除AK本身不存在的情况）。
- **实施步骤**：
  1. **提取被禁用的AK**：根据上述表格，从对应网关日志中提取被拦截的AK（如 `access_id` 或 `AccessKeyId`）。
  2. **确认AK状态**：登录PCM管控台或通过PCM内部查询接口，查询该AK的当前状态，确认是否处于“禁用”状态。
  3. **应急处置**：
     - **若为误禁用/业务正常需求**：联系PCM管理员或安全团队，在PCM系统中将该AK的状态恢复为“启用”。
     - **若为安全合规禁用（如AK泄露触发自动拦截）**：联系业务方在RAM/PCM重新生成新的AK/SK，在业务代码或配置中更新替换，并废弃旧AK，同时排查泄露原因。
- **结果验证**：业务方使用恢复后的AK或新AK重新发起API请求，检查网关日志状态码是否恢复为200，业务调用是否恢复正常。

## 四、非本产品排查
● **RAM/STS产品组**：若日志中提示 `InvalidAccessKeyId`，但PCM系统中该AK状态为“启用”，则不属于PCM拦截问题。需联系RAM/STS团队确认AK是否在RAM层面被删除或禁用。
● **业务方/调用方**：若AK状态均正常，需业务方排查AK是否拼写错误，或是否使用了错误环境（如生产环境使用了测试环境AK）的凭证。
● **对应云产品组**：若ASAPI日志中 `errorCode` 非 `accesskeyid.error`，而是其他业务参数错误（如 `InvalidParameter`），需转交对应云产品（如ECS、VPC等）排查业务逻辑。

## 五、快速定位工具
● **工具/脚本位置**：SLS统一日志查询平台。
● **使用方法**：在SLS控制台中选择对应的Logstore，使用以下查询语句快速统计和定位被拦截的AK：
  - **OSS拦截查询**：
    ```sql
    error_code: InvalidAccessKeyId and status: 403 | select access_id, count(1) as cnt group by access_id order by cnt desc
    ```
  - **SLS拦截查询**：
    ```sql
    Status: 401 | select AccessKeyId, count(1) as cnt group by AccessKeyId order by cnt desc
    ```
  - **ASAPI拦截查询**：
    ```sql
    errorCode: asapi.server.request.parameter.accesskeyid.error | select accessKeyId, count(1) as cnt group by accessKeyId order by cnt desc
    ```
