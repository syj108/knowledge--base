# 告警-风险-异常汇总表

当遇到访问报错，怀疑是[[PCM/PCM/index|PCM]]禁用AK导致的，优先通过拦截日志判定，提取日志中的请求AK，并通过PCM服务查询AK状态。如果已经禁用，采用应急处置方案进行处置，并反馈研发侧排查原因。

## 告警-风险-异常汇总表

| 告警名 | 级别 | 识别方式 | 所属组件 | 触发条件 | 含义 | 应急手册链接 | 是否有EOCC/KB |
| --- | --- | --- | --- | --- | --- | --- | --- |
| PCM与应用双挂且SDK缓存丢失 | 致命 | 应用启动失败/凭证获取失败报错/业务中断告警 | PCM SDK / PCM Core | PCM服务宕机且应用重启导致SDK本地缓存丢失 | SDK无法从PCM获取凭证且本地无缓存，导致应用无法获取有效凭证，业务中断。 | 优先恢复PCM服务，或使用老凭证应急脚本进行应急恢复 | - |
| UMM账户AK上限打满 | 高 | UMM账户AK数量监控/创建派生AK失败日志 | PCM Controller / UMM | 派生AK队列级别配置为ClusterName级别，且存在多集群叠加 | 每个集群为同一个底表AK创建独立队列，多集群叠加把账户AK上限打满，导致无法创建新的派生AK。 | 将派生AK队列级别修改为推荐的 initAK 级别 | - |
| 平台AK访问日志不可行 | 中 | PCM Controller日志/轮转状态监控 | PCM Controller | 平台AK访问日志不可行（当前状态） | PCM无法确认即将禁用的派生AK是否仍有产品在调用，将在第一把队列即将禁用时停止轮转以保护凭证。 | - | - |
| PCM Core服务异常或宕机 | 中 | PCM Core进程监控/健康检查失败/SDK降级日志 | PCM Core | PCM Core进程挂掉、网络不通或超时 | Core宕机后，末期过期老凭证行为暂停，SDK降级返回上次获得的老凭证（未在窗口期末尾）或本地缓存凭证，业务无影响。 | - | - |
| PCM初始化服务异常 | 低 | SDK初始化报错日志 | PCM SDK | 新部署时PCM Core未ready，或产品独立升级时PCM未ready | PCM服务不可用，SDK容错降级，将入参（底表AK）作为凭证返回，或返回最近一次从服务端获取的缓存凭证，业务无影响。 | - | - |
| OSS AK禁用拦截 | - | 日志包含 `"error_code": "InvalidAccessKeyId"` 且 `"status": "403"` | OSS | PCM禁用AK | AK被禁用导致OSS访问请求被拦截 | 参考PCM应急处置方案 | - |
| SLS AK禁用拦截 | - | 日志包含 `"Status": "401"`，或 `"ErrorCode": "Unauthorized"` 且 `"ErrorMsg": "AccessKeyId is disabled"` | SLS (INNER/PUB) | PCM禁用AK | AK被禁用导致SLS访问请求被拦截 | 参考PCM应急处置方案 | - |
| ASAPI AK禁用拦截 | - | 日志包含 `"errorCode": "asapi.server.request.parameter.accesskeyid.error"` 且提示 `The Access Key is disabled` | ASAPI | PCM禁用AK | AK被禁用导致ASAPI接口调用失败 | 参考PCM应急处置方案 | - |
| KMS AK禁用拦截 | - | 日志包含 `"error_code": "Forbidden.AccessKey"` 且 `"error_message": "This AccessKey is not enabled."`，`"status_code": "403"` | KMS | PCM禁用AK | AK被禁用导致KMS访问请求被拦截 | 参考PCM应急处置方案 | - |
| ODPS 访问异常 | - | 参考ODPS aggregated_log 日志示例进行排查 | ODPS | PCM禁用AK | AK被禁用导致ODPS访问异常 | 参考PCM应急处置方案 | - |

> **注**：部分AK禁用拦截告警的级别、具体的应急手册URL链接以及EOCC/KB信息暂未明确，故相关字段以“-”或通用说明填充。
