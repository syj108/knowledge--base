# 数据表结构

- **服务 (Service)**：baseService-umm-ak
- **数据库实例**：ummak
- **数据库名**：ummak

### accesskey_table (AK信息表)

用于存储AK的状态及基本信息，每个 UID 下最大支持 1000 把有效 AK。

| 字段名 | 说明 | 备注/枚举值 |
| --- | --- | --- |
| `access_id` | AK ID | 唯一标识 |
| `access_key` | SK (Secret Key) | |
| `user_id` | 账号 UID | |
| `enabled_flag` | 启用状态 | `1`: 启用，`0`: 禁用 |
| `hidden_flag` | 隐藏状态 | `0`: 不隐藏 |
| `deleted_flag` | 删除状态 | `0`: 未删除，`1`: 已删除 |
| `modified_time` | 修改时间 | UNIX 时间戳 |

## PCM 数据库 (pcm_db)

- **服务 (Service)**：certificate-lifecycle-manager-server
- **数据库实例**：clm_db
- **数据库名**：pcm_db

### init_ak_info (底表AK信息表)

用于存储 PCM 托管的底表 AK 信息。当派生 AK 超过 14 把时，UMMAK 侧会执行删除操作，但此表会保留派生 AK 记录。

| 字段名 | 说明 | 备注/枚举值 |
| --- | --- | --- |
| `access_key_id` | AK ID | |
| `umm_ak_status` | UMM AK 状态 | `0`: 禁用 |
