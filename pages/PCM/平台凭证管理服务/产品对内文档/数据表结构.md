# 数据表结构

### 数据库实例信息
- **所属服务**：baseService-umm-ak
- **数据库实例**：ummak
- **数据库名**：ummak

### accesskey_table 表
- **表说明**：存储 AK（Access Key）的基础信息及状态，包括 initAK、底表AK 和派生AK。
- **核心字段**：
  - `access_id`：AK ID
  - `access_key`：SK（Secret Key）
  - `user_id`：账号 UID
  - `enabled_flag`：启用状态（1 为启用，0 为禁用）
  - `hidden_flag`：隐藏状态（0 为不隐藏）
  - `deleted_flag`：删除状态（1 为已删除，0 为未删除）
  - `modified_time`：修改时间（Unix 时间戳）

## PCM 数据库

### 数据库实例信息
- **所属服务**：certificate-lifecycle-manager-server
- **数据库实例**：clm_db
- **数据库名称**：pcm_db

### init_ak_info 表
- **表说明**：存储 PCM 托管的底表 AK 信息。当白屏未查询到派生 AK 时（如超过14天被 ummak 侧删除），可通过该数据库保留的记录进行查询。
- **核心字段**：
  - `access_key_id`：AK ID
  - `umm_ak_status`：UMM AK 状态（0 表示已禁用）

### ak_info 表
- **表说明**：主要用于存储和查询派生 AK 的相关信息。在排查“派生 AK 被拦截”等问题时，可通过查询该表来验证派生 AK 的状态与存在性。
- **常用查询示例**：

```sql
-- 进入 clm_db 实例后，切换到 pcm_db 数据库
use pcm_db;

-- 检查指定的派生 AK 是否存在
select * from ak_info where access_key_id='****';
```
