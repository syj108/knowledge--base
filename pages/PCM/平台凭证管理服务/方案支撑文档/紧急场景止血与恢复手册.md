# 紧急场景止血与恢复手册

在平台凭证管理服务（PCM）运行过程中，针对各类异常和灾难场景，系统设计了相应的容错降级机制与应急恢复手段。

### 常规异常场景与自动降级

对于大部分常规异常场景，PCM SDK 具备自动容错降级能力，不会对业务造成影响：

| 故障场景 | SDK 降级行为 | 业务影响 |
| --- | --- | --- |
| 新部署时 PCM Core 还未 ready | 将入参作为返回 | 无影响（Core 未禁用老 AK） |
| 运行时 PCM Core 挂了 | 返回上次获取的老凭证（未在窗口期末尾） | 无影响 |
| 产品独立升级，PCM 未 ready | 将入参作为返回 | 无影响 |
| PCM 和应用都挂了需重拉（SDK 缓存未丢失） | 返回上次获取的老凭证 | 无影响 |

### 灾难场景应急操作（SDK 缓存丢失）

当发生 **PCM 和应用都挂了需重拉，且 SDK 缓存丢失** 的极端灾难场景时，SDK 无法自动降级，将导致**业务中断**。

**紧急止血与恢复操作：**
* **恢复 PCM 服务**：优先排查并紧急恢复 PCM 服务端，使应用重新具备获取派生凭证的能力。
* **使用老凭证应急脚本**：若 PCM 服务短时间内无法恢复，需立即使用**老凭证应急脚本**进行止血，保障业务快速恢复。
