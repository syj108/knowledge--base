# 典型问题排查解决方案

**场景1：云产品组件获取派生凭证失败，降级使用底表AK**
*   **问题现象**：云产品组件（Java/Go/Python等）日志中出现获取派生AK失败告警，SDK返回状态码 `code=401`（降级返回底表AK）或 `code=301/302`（返回本地加密文件缓存凭证）。用户可能反馈凭证轮换未生效或收到安全合规告警。
*   **适用范围**：所有接入 PCM SDK/CLI 的云产品组件，涉及 PCM Core、PCM Controller、UMM 等依赖服务异常或网络不通的场景。

**场景2：派生AK队列停止轮转，老化凭证未被禁用**
*   **问题现象**：在 PKM 白屏管控界面或 PCM Controller 日志中，发现某个底表AK的派生AK队列长时间未创建新AK，或最早的老化派生AK未被正常禁用。
*   **适用范围**：PCM Controller 执行凭证生命周期管理和队列轮转的场景。

**场景3：PCM与应用同时宕机重启后业务中断**
*   **问题现象**：PCM 服务与云产品应用同时发生宕机并重新拉起后，应用启动失败或调用云产品API报鉴权失败（业务中断）。
*   **适用范围**：极端故障场景，PCM 服务不可用且 SDK 本地加密缓存文件丢失。

## 二、排查信息收集

*   **必须收集的信息**：
    *   云产品组件信息：实例ID、主机名、应用名称、集群名称（ClusterName）。
    *   凭证信息：底表AK（initAK）、IAMID（格式：`${CLUSTERNAME}:<serverrole名称>` 或 PaaS格式 `{{ .Values.productName }}:{{ .Release.Name }}`）、secretARN。
    *   版本信息：PCM SDK/CLI 版本、PCM Core 版本、PCM Controller 版本。
*   **检查终态的方法**：
    *   登录云产品组件所在容器/主机，检查应用运行日志及 PCM SDK 日志。
    *   登录 PCM Core 容器，检查共享内存缓存状态（initak_queue/endpoint_queue等）及 API 访问日志。
    *   登录 PCM Controller 容器/数据库，检查派生AK队列状态及轮转任务日志。
    *   登录 PKM 白屏管控界面，查看凭证状态及关联日志。
*   **排查问题步骤与关键结果映射**：

| 排查步骤 | 关键排查结果/日志特征 | 对应场景 |
| --- | --- | --- |
| 检查应用端 SDK 日志 | 日志显示 `code=401`，提示“降级返回底表AK” | 场景1：SDK请求Core失败且本地无缓存 |
| 检查应用端 SDK 日志 | 日志显示 `code=301/302`，提示“读取本地加密文件(AES-256-GCM解密)” | 场景1：SDK请求Core失败，但本地文件有有效凭证 |
| 检查 PCM Core 连通性与日志 | Core 容器宕机、网络超时或 API 返回异常 | 场景1：PCM Core 异常导致 SDK 无法获取新凭证 |
| 检查 PCM Controller 轮转日志 | 日志提示“触发最新派生AK保护”或“访问日志不可行/仍在使用” | 场景2：触发队列轮转保护机制，暂停轮转 |
| 检查 SDK 本地缓存文件 | 宿主机/容器内 PCM 本地加密缓存文件不存在或损坏 | 场景3：PCM与应用同挂，且SDK缓存丢失导致业务中断 |

## 三、解决步骤

**场景一：云产品组件获取派生凭证失败，降级使用底表AK**
*   **适用条件**：SDK 日志显示 `code=401` 或 `code=301/302`，且 PCM Core 或 Controller 存在异常。
*   **实施步骤**：
    1.  **确认 PCM Core 状态**：
        *   操作位置：K8s 集群 / 容器管理平台
        *   完整命令：检查 PCM Core Pod 状态 `kubectl get pods -n <pcm-namespace> -l app=pcm-core`。若 Pod 异常，查看日志 `kubectl logs <pod-name> -n <pcm-namespace>`。
        *   结果验证：若 Core 宕机，SDK 会自动降级返回上次获取的老凭证（未在窗口期末尾），业务无影响。需尽快重启或恢复 Core Pod。
    2.  **确认 PCM Controller 与 UMM 状态**：
        *   操作位置：PCM Controller 容器 / UMM 服务
        *   完整命令：检查 Controller 日志中是否有调用 UMM 创建/获取派生AK失败的报错。检查 UMM 服务健康状态。
        *   结果验证：若 UMM 异常，联系 UMM 产品组恢复；若 Controller 异常，重启 Controller Pod。
    3.  **检查网络与认证**：
        *   操作位置：云产品组件容器
        *   完整命令：检查 SDK 请求 Core 时的网络连通性（`curl -v <pcm-core-endpoint>`），确认 PcmToken JWT 签名是否过期或异常。
        *   结果验证：网络恢复或 Token 刷新后，SDK 重新请求 Core 获取新凭证（返回 `code=202/203`）。

**场景二：派生AK队列停止轮转，老化凭证未被禁用**
*   **适用条件**：PKM 白屏管控界面显示派生AK队列未正常轮转，老凭证状态未变更为“已禁用”。
*   **实施步骤**：
    1.  **排查是否触发“产品最新派生AK保护”**：
        *   操作位置：PCM Controller 日志 / PKM 白屏管控界面
        *   完整命令/操作：查看 Controller 轮转任务日志，确认是否提示某产品获取该AK后未再获取新AK。
        *   结果验证：若是此原因，属于正常保护机制。需推动对应云产品组件升级或重新触发凭证获取，待所有产品获取更新AK后，队列自动恢复轮转。
    2.  **排查“平台AK访问日志”状态**：
        *   操作位置：PCM Controller / 日志查询关联模块
        *   完整命令/操作：检查平台AK访问日志是否“不可行”或日志显示该派生AK“仍有调用记录”。
        *   结果验证：若日志不可行，系统会在第一把队列即将禁用时停止轮转；若仍有调用，说明有产品仍在使用老AK。需通过日志查询关联功能定位调用方，推动其重启或刷新凭证。
    3.  **手动灰度禁用（热升级场景）**：
        *   操作位置：PKM 白屏管控界面（运维控制台）
        *   完整命令/操作：对于热升级项目，原始凭证默认不禁用。若需禁用老凭证，在观测日志确认无调用后，通过运维控制台灰度执行禁用操作。
        *   结果验证：老凭证状态变更为“已禁用”，UMM 中对应 AK 被禁用。

**场景三：PCM与应用同时宕机重启后业务中断**
*   **适用条件**：PCM 服务与应用同时重启，应用启动失败或 API 鉴权失败，SDK 缓存丢失。
*   **实施步骤**：
    1.  **确认 SDK 缓存状态**：
        *   操作位置：云产品应用容器/主机
        *   完整命令/操作：检查本地加密文件（AES-256-GCM加密）是否存在。若不存在，确认缓存丢失。
        *   结果验证：确认因缓存丢失导致无法降级。
    2.  **优先恢复 PCM 服务**：
        *   操作位置：K8s 集群 / 容器管理平台
        *   完整命令/操作：优先拉起 PCM Core 和 PCM Controller Pod，确保 PCM 服务 Ready。
        *   结果验证：PCM 恢复后，应用 SDK 自动请求 Core 获取新凭证，业务恢复。
    3.  **使用老凭证应急脚本（PCM 无法短时恢复时）**：
        *   操作位置：云产品应用容器/主机
        *   完整命令/操作：执行老凭证应急脚本，将底表AK（initAK）临时注入应用配置或环境变量中。
        *   结果验证：应用使用底表AK恢复业务。待 PCM 恢复后，需清理应急配置，切回 PCM 托管模式。

## 四、非本产品排查

*   **UMM（AK生命周期管理）异常**：若排查确认 PCM Controller 调用 UMM 创建/获取派生AK失败，且 UMM 接口返回错误，不属于 PCM 问题，需联系 **UMM 产品组** 协助排查 AK 存储与生命周期管理异常。
*   **AAS（账户管理服务）异常**：若 UMM 关联账户凭证失败，AAS 返回账户数据异常，需联系 **AAS 产品组** 协助排查平台账户统一管理问题。
*   **标准网关/AK私用网关鉴权失败**：若 PCM 已成功派发派生AK，但云产品调用 POP、OpenAPI、OSS 等标准网关或 AK 私用网关时仍报鉴权失败，需联系 **对应网关产品组** 排查网关侧的 AK 签名校验逻辑。

## 五、快速定位工具

*   **PKM 白屏管控界面**：
    *   位置：运维控制台 -> PCM 凭证管理
    *   使用方法：用于可视化查看凭证状态（轮换中/已禁用/正常）、派生AK队列级别（initAK级别/ClusterName级别）、管控模式（None/CompatibilityMode/StrictMode/initStrictMode），以及关联 AK 使用记录日志。
*   **PCM SDK/CLI 诊断命令**：
    *   位置：云产品组件容器内
    *   使用方法：使用 PCM CLI 工具执行凭证诊断命令，检查内存缓存、本地文件缓存状态及与 PCM Core 的连通性。
